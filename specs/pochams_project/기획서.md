## 1. 프로젝트 개요

포챔스 배틀은 **포켓몬 챔피언스** 게임 플레이어를 위한 종합 전략 플랫폼이다.  
샘플(세부 육성 구성) 공유, 팀 빌딩, 대미지 계산, 사용률 통계 기능을 하나의 사이트에서 제공한다.

### 핵심 가치
- 비로그인으로도 샘플 등록·팀 빌딩 가능 (익명 참여 허용)
- 커뮤니티 좋아요 기반 인기 샘플 큐레이션
- OCR로 스위치 캡쳐사진을 찍어 샘플 자동 등록
- 정확한 대미지 계산기 (Gen 9 규칙 적용)

## 2. Pokenova와의 관계

### 2-1. 데이터 공유 구조

```
pokenova_project/data/          pochams_project (DB)
─────────────────────────       ──────────────────────────
quiz_pokemon.json       ──seed──▶  pokemon_data 테이블
quiz_moves.json         ──seed──▶  move_data 테이블
quiz_items.json         ──seed──▶  item_data 테이블
quiz_abilities.json     ──seed──▶  ability_data 테이블
learnsets.json          ──seed──▶  pokemon_learnset 테이블
```

- **pokenova가 원천 데이터 관리자.** 포켓몬·기술·도구·특성·배우는기술 데이터는 pokenova 레포의 JSON 파일에서 관리한다.
- **pochams는 그 데이터를 DB에 seed해서 사용.** `npm run seed` 한 번으로 모든 마스터 데이터가 갱신된다.
- 두 프로젝트는 **Neon PostgreSQL 하나를 공유**한다. DB 수정은 pokenova JSON → seed 재실행 순서로만 한다.

### 2-2. 역할 분리

| 항목 | pokenova | pochams |
|---|---|---|
| 역할 | 포켓몬 도감 + 퀴즈 플랫폼 | 팀 빌딩 + 샘플 공유 플랫폼 |
| 마스터 데이터 관리 | ✅ JSON 파일로 관리 | ❌ (읽기만) |
| 유저 계정 | ❌ | ✅ NextAuth |
| 커뮤니티 기능 | ❌ | ✅ 좋아요, 팀, 샘플 |
| 계산기 | ❌ | ✅ 대미지 계산기 |
| 배포 플랫폼 | Vercel (pokenova.pochams.com) | Vercel (pochams.com) |

### 2-3. 챔피언스 전용 필터

pokenova의 포켓몬·기술 데이터에는 `in_champions` 플래그가 있다.  
pochams는 이 플래그가 `true`인 데이터만 사용해 포켓몬 챔피언스 범위로 제한한다.  
단, 관련 포켓몬(파트너/카운터) 태그 입력에는 메가진화폼도 포함된다(`includeMega=true`).

## 3. 주요 기능

### 3-1. 샘플 시스템
- **샘플**: 포켓몬 1마리의 완성된 육성 구성 (특성·성격·도구·기술·노력치·테라타입·메모·파트너·카운터)
- **샘플 이름 형식**: `[별명(선택)] {포켓몬이름} 샘플` — 접미사 고정·수정 불가, 별명만 입력 가능
  - 영어 모드 표시: `[nickname] {EnglishName} Set`
  - DB 저장은 항상 한국어 형식, 영어 표시는 런타임 변환 (`buildSampleDisplayName`)
- 비로그인 익명 등록 가능 (익명 표시명 자동 생성)
- 좋아요 기반 인기도 산출
- **댓글 기능**: 로그인 유저만 댓글 작성 가능 (`StandaloneSampleComment` 모델)
- 포맷(싱글/더블) 구분
- 파트너/카운터 포켓몬 태그 — 메가진화폼 포함, 화살표 키 네비게이션, Enter 선택

### 3-2. 팀 빌딩
- 최대 6마리 팀 구성
- 팀 내 각 포켓몬에 샘플 연결
- 렌탈 코드 10자리 기록 가능
- **팀 작성 폼에서 메가스톤 선택 시 해당 메가폼으로 자동 전환** (SampleCard 양방향 연동)
- **팀 상세 페이지에서 base 폼 + 메가스톤 조합 시 메가폼 이름·스프라이트로 올바르게 표시**

### 3-3. OCR 자동 등록
- 스위치 캡쳐사진 업로드 → EasyOCR 서버(Railway 배포)가 포켓몬 이름·기술·노력치 자동 인식
- 인식 결과를 learnset 기반으로 필터링해 오매칭 최소화
- 매칭 실패 시 항상 가장 유사한 실제 DB 데이터로 fallback (빈값 반환 없음)
- 이미지 회전 버튼 (90도씩), 미리보기 크기 확대
- 노력치 합계 표시 (최대 66), 소요시간 안내 문구
- Railway 외부 서버와 HTTP 통신 (`/api/ocr` → `OCR_SERVER_URL`)

### 3-4. 대미지 계산기
- Gen 9 규칙 적용
- 공격자 / 수비자 포켓몬 선택, SP 배분, 성격, 랭크 보정, 도구
- 날씨 보정: 쾌청·비·모래바람·설경
- 필드 보정: 그래스·미스트·사이코·일렉트릭 (타입 강화 + 미스트의 드래곤타입기술 약화, 그래스의 지진·땅고르기 약화)
- 벽 보정: 리플렉터·빛의장막·오로라베일 (급소 시 무효)
- 솔라빔·솔라블레이드: 쾌청 이외 날씨 위력 절반
- 무조건 급소 기술 자동 체크
- 특성 보정: 약 60종 이상 반영 (특성 이름은 DB 포켓몬 데이터에서 가져옴)
- 멀티스케일: 수비자 특성으로 멀티스케일 선택 시 체크박스 표시 → 활성화하면 받는 데미지 ×0.5 계산
- **보정값 칩 표시**: 계산 결과에 적용된 모든 보정(자속보정·타입상성·날씨·필드·벽·급소·멀티스케일 등)을 컬러 칩(pill)으로 표시 — 강화(배율 >1) emerald, 약화(배율 <1) orange
- 탁쳐서떨구기: 수비자가 도구를 지니고 있으면 위력 ×1.5 (수비자 도구 선택 시 자동 적용)
- 바디프레스: 공격자 방어 수치로 계산
- 지닌도구 지원, 공격자/수비자 스왑 시 도구 교환
- Combobox 화살표 키 네비게이션
- 샘플 불러오기로 등록된 샘플의 스탯 자동 적용
- 실수치 공식: HP = 종족치+75+SP, 나머지 = floor((종족치+20+SP) × 성격보정)
- **수비자 UI 아이템/특성 순서**: 아이템 먼저, 특성 나중 표시
- **일본어 모드에서 특성명**: `/api/data/abilities` API에서 ja 필드를 가져와 일본어로 표시
- 기술·도구·포켓몬 명칭은 모두 DB 데이터 기준으로만 사용 (하드코딩 금지)

### 3-5. 사용률 통계
- 샘플 등록/수정/삭제 시 자동 집계 (`UsageStat` 테이블)
- countBased(샘플 수) / weightedBased(좋아요 수) 두 가지 기준
- 포켓몬별 인기 기술·도구·특성·노력치 Top 3 표시
- 코어 조합(자주 함께 쓰는 포켓몬) 분석

### 3-6. 패치노트 시스템
- **PatchNote 모델**: `title`, `content`, `category(String?)`, `createdAt`
- **카테고리**: `"feature"` (기능추가 · 초록) / `"bug"` (버그수정 · 빨강) / `null` (미분류 · 기본색)
- **메인 패널** (`PatchNotePanel`): 최신 5개 표시, 하단 "전체 패치노트 보기 →" 링크
- **전체 패치노트 페이지** (`/patch-notes`): 아코디언 형식, 제목(좌) + 날짜·화살표(우), 1줄 설명은 Y축 중앙정렬, 10개씩 페이지네이션
- **캐시**: `Cache-Control: no-store` — 관리자 수정 즉시 반영 필수
- **관리자 UI** (`PatchNoteManager`): 카테고리 토글 버튼(기능추가/버그수정/선택안함)으로 등록·수정 가능

### 3-7. 버그 신고 시스템
- 신고 카테고리: 포켓몬 정보 오류·기술 정보 오류·도구 오류·계산기 오류·페이지 오류·비속어 신고
- 비로그인·로그인 모두 신고 가능, 한국어 비속어 필터 적용
- 관리자 처리 상태: open / 처리중 / 완료 / 보류

## 10. 다국어(i18n) 언어 전환 기능

> **목적:** 해외 유저 접근성 향상 — 브라우저 언어 자동 감지, KO/JA/EN 3개 토글 버튼 제공  
> **상태:** ✅ KO/EN 완료(main) + JA 전 페이지 완료(feature/japanese 브랜치 — main 머지 예정)  
> **도메인:** https://pochams.com

### 10-1. 구현 방식

**브랜치:** feature/japanese (일본어 개발 중 — main 미머지)

**언어 결정 우선순위:**
1. URL 파라미터 `?lang=en` / `?lang=ko` / `?lang=ja` (공유 링크 최우선)
2. localStorage 저장값 (재방문 시 이전 선택 유지)
3. `navigator.language` 브라우저 언어 감지 (ko → 한국어, ja → 일본어, 나머지 → 영어)

**핵심 파일:**
| 파일 | 역할 |
|---|---|
| `src/contexts/LangContext.tsx` | React Context — `lang`, `setLang`, 감지 로직 |
| `src/lib/i18n.ts` | UI 텍스트 번역 사전 (nav, 버튼, 섹션 제목 등) |
| `src/components/shared/T.tsx` | `<T ko="한국어" en="English" />` 인라인 번역 컴포넌트 |
| `src/components/shared/Header.tsx` | KO / JA / EN 3개 토글 버튼 |
| `src/app/layout.tsx` | LangProvider 래핑, `<html lang>` 동적 반영 |

### 10-2. 데이터 필드 전환

DB에 `ko`/`en`/`ja` 필드가 존재한다. 언어에 따라 전환:

```tsx
const { lang } = useLang()
pokemon[lang]  // 'ko' → "리자몽", 'en' → "Charizard", 'ja' → "リザードン"
```

**샘플 이름 다국어 변환 유틸:** `src/lib/nameMaps.ts` — `buildSampleDisplayName(sampleName, pokemonKo, pokemonEn, lang, pokemonJa?)` 함수로 DB 저장된 한국어 이름을 런타임에 영어/일본어 형식으로 변환.  
- 영어: `"{영어이름} Set"` 형식  
- 일본어: `"{일본어이름} サンプル"` 형식  
- 한국어: DB 저장값 그대로

**일본어 명칭 소스:**
| 분류 | 출처 |
|---|---|
| 포켓몬 이름 | PokeAPI `ja-Hrkt` 필드 (katakana) |
| 기술·도구·특성 | PokeAPI `ja-Hrkt` 필드 |
| 성격 | `nameMaps.ts` NATURE_JA 수동 정의 |
| 타입 | `nameMaps.ts` TYPE_JA 수동 정의 |
